QUICK START
-----------

1. Initialize Git (if not already):
   git init

2. Add files (respecting .gitignore):
   git add .

3. Check what will be committed:
   git status

4. Commit:
   git commit -m "Initial commit"

5. Add remote:
   git remote add origin <your-repo-url>

6. Push:
   git push -u origin main

COMMON COMMANDS
---------------

Check ignored files:
  git status --ignored

Check if file is ignored:
  git check-ignore -v filename

Remove already tracked file:
  git rm --cached filename
  git commit -m "Remove tracked file"

Clean ignored files:
  git clean -fdX

TROUBLESHOOTING
---------------

Problem: File already tracked
Solution: git rm --cached filename

Problem: node_modules showing
Solution: git rm -r --cached node_modules/

Problem: .env committed
Solution: git rm --cached .env

BEST PRACTICES
--------------
1. Add .gitignore before first commit
2. Review git status before committing
3. Never commit sensitive data (.env, keys)
4. Keep .gitignore updated
5. Use .env.example for templates
